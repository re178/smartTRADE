Public API Documentation
🔐 Authentication

All requests must include the following headers:

  • X-API-Key: <your_api_key>
  • X-API-Secret: <your_api_secret>

If invalid or missing, you will receive a 401 Unauthorized. Disabled keys return 403 Forbidden.

📡 Base URL

https://your-rts-server.com/api/public

Product‑Aware: The API automatically uses the trading product (MT5 or Deriv) that you have selected in the dashboard. All data is returned in real‑time where supported.

🔑 Permission Model

When creating an API key, you assign a set of permissions. Each permission grants access to a group of endpoints. The following permissions are available:

PermissionWhat it allowsEndpoints affected
account.read View account balance, equity, margin, etc. GET /account
positions.read View open positions and trades. GET /positions, GET /trades
market.read Access real‑time and historical market data (prices, candles, symbols, capabilities, signals). GET /prices, GET /price/:symbol,
GET /candles, GET /symbols,
GET /capabilities, GET /signal
history.read View trade history (closed trades). GET /history
signals.write Submit new trade signals (store them). POST /signals
orders.write Place, close, modify, and cancel orders. Also list pending orders. POST /orders, POST /close/:tradeId,
POST /auto-trade, GET /pending-orders,
DELETE /order/:orderId

Note: If a key does not have the required permission, the API returns 403 Forbidden with an error message. The permission checks are performed after authentication, so you'll always get a clear indication if you're missing the right permission.

🧩 Common Use Cases (step‑by‑step examples)
📈 1. Get account balance and open positions

Use this to display your current trading status before making decisions.

# Get account
curl -H "X-API-Key: rts_pk_..." -H "X-API-Secret: rts_sk_..." \
     https://your-rts-server.com/api/public/account

# Get open positions
curl -H "X-API-Key: rts_pk_..." -H "X-API-Secret: rts_sk_..." \
     https://your-rts-server.com/api/public/positions
💹 2. Get real‑time prices for multiple symbols

Use this to update your UI or feed into a trading algorithm.

curl "https://your-rts-server.com/api/public/prices?instruments=EUR_USD,GBP_USD,USD_JPY" \
     -H "X-API-Key: rts_pk_..." -H "X-API-Secret: rts_sk_..."
📉 3. Fetch OHLCV candles for backtesting

Get historical data to test your strategies. Supports M1, M5, M15, M30, H1, H4, D.

curl "https://your-rts-server.com/api/public/candles?pair=EUR_USD&count=100&granularity=M5" \
     -H "X-API-Key: rts_pk_..." -H "X-API-Secret: rts_sk_..."
📊 4. Get symbol metadata (pip, leverage, min/max lot)

Use this to calculate position sizes, stop‑loss distances, and manage risk per symbol.

curl -H "X-API-Key: rts_pk_..." -H "X-API-Secret: rts_sk_..." \
     https://your-rts-server.com/api/public/symbols
🚀 5. Place a market order

Execute a trade immediately at the current market price.

curl -X POST https://your-rts-server.com/api/public/orders \
     -H "Content-Type: application/json" \
     -H "X-API-Key: rts_pk_..." \
     -H "X-API-Secret: rts_sk_..." \
     -d '{"symbol":"EURUSD", "type":"buy", "volume":0.1}'
🧠 6. Generate a trading signal

Use a strategy to get a signal (BUY/SELL) with entry, SL, TP.

curl "https://your-rts-server.com/api/public/signal?pair=EUR_USD&strategy=rsi" \
     -H "X-API-Key: rts_pk_..." -H "X-API-Secret: rts_sk_..."
⚠️ Error Handling

The API returns standard HTTP status codes:

  • 200 – Success (with JSON body).
  • 400 – Bad request (e.g., missing parameters).
  • 401 – Unauthorised (invalid or missing API key/secret).
  • 403 – Forbidden (key disabled or insufficient permissions).
  • 404 – Not found (e.g., unknown symbol).
  • 429 – Too many requests (if rate limiting is enabled).
  • 500 – Internal server error.

Error responses include a JSON error field with a description.

⏱️ Rate Limiting

Currently, there is no hard rate limit applied, but we recommend keeping requests under 10 per second to avoid server overload. In the future, rate limits may be enforced and communicated via the X-RateLimit-* headers.
