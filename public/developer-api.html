Public API Documentation
🔐 Authentication

All requests must include the following headers:

  • X-API-Key: <your_api_key>
  • X-API-Secret: <your_api_secret>

If invalid or missing, you will receive a 401 Unauthorized. Disabled keys return 403 Forbidden.

📡 Base URL

https://your-rts-server.com/api/public

Product‑Aware: The API automatically uses the trading product (MT5 or Deriv) that you have selected in the dashboard. All data is returned in real‑time where supported.
📊 Endpoints
Method Endpoint Description Permission
GET /account Latest account info (balance, equity, margin) account.read
GET /positions All open positions positions.read
GET /trades All open trades (alias for positions) positions.read
GET /prices?instruments=... (comma‑separated) Real‑time bid/ask prices for one or more symbols market.read
GET /price/:symbol Real‑time price for a single symbol (e.g., /price/EUR_USD) market.read
GET /candles?pair=...&count=...&granularity=... OHLCV candles for backtesting and technical indicators. Granularity: M1, M5, M15, M30, H1, H4, D market.read
GET /symbols Metadata for all trading symbols (pip, leverage, min/max lot, etc.) market.read
GET /capabilities Broker capabilities (supportsMarketOrders, supportsLimitOrders, etc.) market.read
GET /history Trade history (closed trades) history.read
GET /signal?pair=...&strategy=... Generate a trading signal based on a strategy (sma, rsi, macd, etc.) market.read
POST /signals Submit a trade signal (store it in the system) signals.write
POST /orders Place a market order (requires symbol, type = buy/sell, volume) orders.write
POST /close/:tradeId Close an open trade by its ID orders.write
POST /auto-trade Auto‑trade based on a strategy signal (uses risk %) orders.write
GET /pending-orders List pending orders (if broker supports them) orders.write
DELETE /order/:orderId Cancel a pending order orders.write
DELETE /history Delete all closed trades from history (cleanup) history.write (admin‑like)
📨 Example Request (cURL)
curl -H "X-API-Key: rts_pk_abc123..." \
     -H "X-API-Secret: rts_sk_def456..." \
     https://your-rts-server.com/api/public/account
📥 Example POST /orders
curl -X POST https://your-rts-server.com/api/public/orders \
  -H "Content-Type: application/json" \
  -H "X-API-Key: rts_pk_..." \
  -H "X-API-Secret: rts_sk_..." \
  -d '{"symbol":"EURUSD", "type":"buy", "volume":0.1}'
📥 Example GET /candles
curl "https://your-rts-server.com/api/public/candles?pair=EUR_USD&count=50&granularity=M5" \
  -H "X-API-Key: rts_pk_..." \
  -H "X-API-Secret: rts_sk_..."
📥 Example GET /symbols
curl -H "X-API-Key: rts_pk_..." \
     -H "X-API-Secret: rts_sk_..." \
     https://your-rts-server.com/api/public/symbols

Replace the domain with your actual RTS server address. The granularity supports M1, M5, M15, M30, H1, H4, D.
